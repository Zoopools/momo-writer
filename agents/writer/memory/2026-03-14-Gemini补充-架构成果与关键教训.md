## 🚨 关键教训（精炼版）

### 教训 1: 公司端 plist 命名

**事实**: 公司端 plist 名为 `com.user.agfs.plist`，**非** `server` 后缀

**家里端**: `com.user.agfs.plist`  
**公司端**: `com.user.agfs.plist` （实际部署后发现）

**注意**: 之前记录的 `com.user.agfs.server.plist` 是错误信息，实际两端均为 `com.user.agfs.plist`

**修正**: 运维脚本应检测的实际文件名：
```bash
# 正确的检测逻辑
if [ -f "$HOME/Library/LaunchAgents/com.user.agfs.plist" ]; then
    PLIST_FILE="$HOME/Library/LaunchAgents/com.user.agfs.plist"
fi
```

### 教训 2: 守护进程与端口占用

**事实**: 守护进程会自动拉起进程，导致端口占用

**场景**:
```bash
# 1. LaunchAgent 已加载并运行
launchctl load ~/Library/LaunchAgents/com.user.agfs.plist
# → AGFS Server 自动启动

# 2. 尝试手动启动（错误！）
~/.local/bin/agfs-server -c config.yaml
# → bind: address already in use
```

**根本原因**: LaunchAgent 的 `KeepAlive` 特性会自动重启进程

**正确操作**:
```bash
# 修改配置前必须先 unload
launchctl unload ~/Library/LaunchAgents/com.user.agfs.plist

# 修改配置...

# 重新加载
launchctl load ~/Library/LaunchAgents/com.user.agfs.plist
```

**禁止**:
- ❌ 在 LaunchAgent 加载时手动启动
- ❌ 使用 nohup 与 LaunchAgent 同时运行
- ❌ 直接 kill 进程（KeepAlive 会自动重启）

### 教训 3: mkdir -p 是日志重定向的前提

**事实**: 物理目录 `mkdir -p` 是所有日志重定向的前提

**错误场景**:
```bash
# 目录不存在时直接重定向
~/.local/bin/agfs-server > ~/Documents/openclaw/logs/agfs.server.log 2>&1
# → No such file or directory
```

**正确操作**:
```bash
# 必须先创建目录
mkdir -p ~/Documents/openclaw/logs

# 然后重定向
~/.local/bin/agfs-server > ~/Documents/openclaw/logs/agfs.server.log 2>&1
```

**强制规范**:
```bash
# 所有涉及日志的操作必须遵循：
ensure_dir() {
    local dir="$1"
    if [ ! -d "$dir" ]; then
        mkdir -p "$dir" || {
            echo "❌ 无法创建目录: $dir"
            exit 1
        }
    fi
}

# 使用
ensure_dir ~/Documents/openclaw/logs
~/.local/bin/agfs-server > ~/Documents/openclaw/logs/agfs.server.log 2>&1
```

### 三大铁律

1. **命名检测**: 部署前必须确认实际文件名，禁止假设
2. **进程管理**: LaunchAgent 管理时禁止手动干预
3. **目录先行**: 所有文件操作必须先确保目录存在
