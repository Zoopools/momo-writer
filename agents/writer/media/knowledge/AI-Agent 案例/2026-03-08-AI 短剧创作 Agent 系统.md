## 🎯 项目定位

**这不是游戏加速器**，是 **AI 短剧创作 Agent 系统**！

**核心功能**: AI 视频分镜师团队（跑在 Claude Code 里）

**应用领域**: AI 视频创作（短剧/视频生成）

**技术栈**: Claude Code + Seedance 2.0

**商业价值**: 变现工具（可直接用于赚钱）

---

## 🏗️ 4 角色 Agent 架构

| 角色 | 职责 | 产出文件 |
|------|------|---------|
| **总编剧**（主 Agent） | 全局调度，流程控制 | 流程协调 |
| **视觉指导**（Sub Agent 1） | 拆戏、讲戏、质量审核 | `outputs/ep01/visual-direction.md` |
| **美术指导**（Sub Agent 2） | 人物/场景参考图提示词 | `assets/character-prompts.md`、`assets/scene-prompts.md` |
| **执行场记**（Sub Agent 3） | Seedance 2.0 标准提示词 | `outputs/ep01/seedance-prompts.md` |

---

## 🔄 工作流程

```
剧本文本 (script/ep01.md)
  ↓
~start 启动（选择视觉风格/目标媒介）
  ↓
视觉指导分析（拆戏 + 镜头设计）
  ↓
美术指导设计（人物/场景提示词）
  ↓
执行场记编写（Seedance 2.0 视频提示词）
  ↓
输出可用文件 (outputs/ep01/)
```

**命令系统**:
- `~start` - 启动流程
- `~design` - 启动美术设计
- `~prompt` - 生成视频提示词
- `~status` - 查看项目状态
- `~review` - 查看审核详情

## 💡 核心设计亮点

### 1. 角色分工明确

每个 Agent 专注一件事：
- 总编剧：流程控制
- 视觉指导：镜头语言
- 美术指导：视觉设计
- 执行场记：提示词规范化

**优势**: 每个环节质量都比一人包办高

### 2. 文件留档机制

中间成果全部保存为文件：
- 下游角色读文件而不是靠记忆
- 不管做多少集都不会丢信息
- 中途断了也没关系，`~status` 查看进度

### 3. 流程可控

每一步都可以干预：
- 不是黑箱，跑完给一个结果
- 三个阶段每一步都可以看产出
- 觉得不对就直接说"这个角色衣服再破一些"

### 4. 跨集复用

角色和场景提示词跨集累积：
- 做第二集时老角色直接复用
- 新角色自动追加
- 做十集跟做一集的前期准备量差别不大

## 💬 核心金句

> "分成四个角色分工合作，每个角色只专注自己最擅长的事情。各干各的，每个环节的质量都比一个人包办要高。"

> "它不是一个黑箱，跑完给你一个结果爱用不用。三个阶段每一步你都可以看产出，觉得哪里不对就直接说。"

> "它给你的是 80 分的底子，不是 100 分的成品。AI 生成的内容肯定还需要你过目调整。"
