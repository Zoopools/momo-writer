## 🎯 方案概述

**之前方案**: 独立浏览器模式（每次需要重新登录公众号）

**现在方案**: Chrome 扩展模式（复用哥哥的登录状态）⭐ 推荐

---

## 📋 核心区别

### 旧方案（独立浏览器）

```bash
打开独立 Chrome 实例 → 需要重新扫码登录公众号 → 发布文章
```

**问题**: 每次都要重新登录，很麻烦！

---

### 新方案（Chrome 扩展）

```bash
哥哥已经登录公众号后台 → 附加标签页 → 直接使用登录状态 → 发布文章
```

**优势**:
- ✅ 无需重新登录
- ✅ 使用哥哥的真实账号
- ✅ 所有 Cookie/Session 共享
- ✅ 更安全（哥哥可以手动分离）

---

## 🚀 使用方式

### 方式 1: 全自动发布（推荐）

**哥哥指令**:
```
"小媒，写一篇关于 XXX 的文章并用我的浏览器发布到公众号"
```

**执行流程**:
1. ✅ 创作内容
2. ✅ 生成封面图（baoyu-cover-image）
3. ✅ 排版成微信格式（baoyu-format-markdown）
4. ✅ **使用 Chrome 扩展控制哥哥的浏览器**
   - 打开公众号后台（哥哥已登录）
   - 导航到新建文章页面
   - 填写标题、内容、封面
   - 点击发布
5. ✅ 飞书通知哥哥

**关键配置**:
```javascript
// 使用 Chrome 扩展模式（不是独立浏览器）
profile: "chrome"
targetId: "从 openclaw browser tabs 获取"
```

## ⚠️ 注意事项

### 1. 附加/分离标签页

**附加**（让 OpenClaw 控制）:
- 哥哥点击 Chrome 工具栏扩展图标
- 图标显示 "ON" = 已附加

**分离**（停止控制）:
- 哥哥再次点击扩展图标
- 图标显示 "OFF" = 已分离

### 2. 使用前确认

**每次发布前检查**:
```bash
# 确认有已附加的标签页
openclaw browser --browser-profile chrome tabs
```

**如果没有标签页**:
- 提醒哥哥：「哥哥，请先附加公众号后台标签页」
- 等待哥哥附加后再发布

### 3. 安全考虑

**不要访问**:
- ❌ 哥哥的个人邮箱
- ❌ 银行账户
- ❌ 其他敏感信息页面

**只访问**:
- ✅ 公众号后台
- ✅ 与发布相关的页面
