## 📋 项目概述

**目标**: 建立分布式情报监控系统，每日 09:30 自动生成《游戏热点猎手·早报》，推送至飞书。

**核心理念**: 飞书多维表格作为配置中枢，而非数据存储容器。

---

## 🏗️ 技术架构

```
┌─────────────────────────────────────────────────────────┐
│  飞书多维表格 (配置中枢)                                │
│  ├─ 表 A: 01_资讯源配置 (10-20 条)                     │
│  ├─ 表 B: 02_B 站博主配置 (10-30 条)                   │
│  └─ 表 C: 03_Steam 板块配置 (3 条)                     │
├─────────────────────────────────────────────────────────┤
│  小猎抓取引擎 (每天 09:00 执行)                        │
│  ├─ 资讯源抓取 → web_fetch + 语义去重                  │
│  ├─ B 站博主抓取 → yt-dlp/browser + 热词提取           │
│  └─ Steam 板块抓取 → Browser + Highcharts 数据注入      │
├─────────────────────────────────────────────────────────┤
│  早报生成器 (09:30 推送)                               │
│  ├─ 数据整合 → 关联分析                                │
│  ├─ Markdown 格式化                                    │
│  └─ 飞书消息推送                                       │
└─────────────────────────────────────────────────────────┘
```

## 🔄 执行流程

```
每天 09:00 执行 (早报前 30 分钟)

1. 读取表 A (资讯源配置)
   └─ 筛选：激活=✅ 的记录
   └─ 对每条记录执行抓取 → 语义去重
   └─ 更新"最后抓取"字段

2. 读取表 B (B 站博主配置)
   └─ 筛选：激活=✅ 的记录
   └─ 对每条记录执行抓取 → 筛选 24H 内视频
   └─ 更新"最后抓取"字段

3. 读取表 C (Steam 板块配置)
   └─ 筛选：激活=✅ 的记录
   └─ 对每条记录执行抓取 → 计算增长率
   └─ 更新"最后抓取"字段

4. 生成早报 (09:30)
   └─ 【📡 02 专区·全球雷达】→ 资讯整合
   └─ 【🎥 重点博主·侦察报告】→ 爆款视频
   └─ 【📈 Steam·脉搏跳动】→ 增长告警

5. 推送早报
   └─ 飞书消息 (待配置)
```

## ⚠️ 注意事项

### 速率控制
- **资讯源**: 每站间隔 1-2 秒
- **B 站博主**: 每博主间隔 2-3 秒
- **Steam 板块**: 每游戏间隔 2-3 秒
- **执行频率**: 每天 1 次 (09:00)

### 异常处理
- **抓取失败**: 记录日志，跳过该源
- **数据异常**: 标记告警，人工复核
- **推送失败**: 本地保存，重试机制

### 配置更新
- **新增配置**: 第二天自动生效
- **修改配置**: 第二天自动生效
- **删除配置**: 软删除 (保留历史记录)
