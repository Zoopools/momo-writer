## 🔴 Token 使用约束（防止 <400> 错误）
- 单次输入超过 **100K tokens** → 必须分段处理
- System Prompt + 用户输入总和 < **150K tokens**
- 遇到超长内容先提取核心再处理
- 触发 <400> 错误时自动截断并重试
- 技能调用前检查输入长度

## 🐦 Twitter/X 内容抓取流程（重要 ⭐）

**当用户发送 Twitter/X 链接时，必须执行以下步骤**：

### 步骤 1：识别链接
- 检测链接格式：`https://x.com/...` 或 `https://twitter.com/...`

### 步骤 2：执行抓取命令（必须执行）
```bash
fetch-tweet --url "用户提供的链接"
```

**示例**：
```bash
fetch-tweet --url "https://x.com/cheery9998/status/2031246651524837566?s=46"
```

### 步骤 3：解析结果
执行命令后会返回 JSON 格式数据，提取以下字段：
- `tweet.author` — 作者名称
- `tweet.screen_name` — 用户名
- `tweet.article.title` — 推文标题
- `tweet.article.full_text` — 完整内容
- `tweet.likes` — 点赞数
- `tweet.retweets` — 转发数
- `tweet.views` — 浏览量

### 步骤 4：回复用户
```
✅ 已成功抓取 Twitter 内容！

**作者**: 作者名 (@用户名)
**标题**: 推文标题
**互动**: XX 赞, XX 转发, XX 浏览

**核心内容**:
（提炼的摘要，200字以内）

需要我基于这个内容创作文章吗？
```

### ⚠️ 重要提醒
- **必须执行** `fetch-tweet` 命令，不能跳过
- 命令已在 PATH 中，直接执行即可
- 如返回错误，再使用浏览器截图作为备选
- 不要告诉用户"无法获取"，必须先尝试抓取
