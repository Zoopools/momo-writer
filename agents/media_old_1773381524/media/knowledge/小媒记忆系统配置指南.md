## 🚀 QMD 配置步骤（未来执行）

### 步骤 1: 初始化 QMD

```bash
cd ~/Documents/openclaw/agents/media
qmd init
```

### 步骤 2: 配置索引范围

创建 `qmd.json`:
```json
{
  "collections": {
    "memory": {
      "paths": [
        "memory/*.md",
        "MEMORY.md"
      ]
    }
  }
}
```

### 步骤 3: 建立索引

```bash
qmd update
qmd embed
```

### 步骤 4: 验证检索

```bash
qmd search "公众号发布" -c memory -n 5
```

**预期**: <1 秒响应

## 🖤 小媒的记忆策略

### 当前策略（v1.0）

**基础记忆系统**:
- ✅ 每日记忆（memory/YYYY-MM-DD.md）
- ✅ 长期记忆（MEMORY.md）
- ✅ 按需读取（不一次性读取所有）

**优势**:
- ✅ 简单直接
- ✅ 不会超长报错
- ✅ 易于维护

**劣势**:
- ❌ 无法快速检索
- ❌ 没有结构化知识
- ❌ 记忆质量不高

---

### 未来策略（v2.0）

**QMD 记忆检索**:
- ✅ 快速检索（<1 秒）
- ✅ 按需检索（不读取所有文件）
- ✅ 关键词匹配

**记忆评分**:
- ✅ 复用价值（0-3 分）
- ✅ 时效性（0-3 分）
- ✅ 可信度（0-3 分）
- ✅ 总分 >=6 才沉淀

**Ontology 图谱**:
- ✅ 结构化知识
- ✅ 关系推理
- ✅ 跨技能共享
