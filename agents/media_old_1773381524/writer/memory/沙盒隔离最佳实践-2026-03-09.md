## ❌ 核心错误

### 1. 沙盒没有真正隔离

**错误做法**:
```bash
# ❌ 复制生产配置到沙盒
cp -r ~/.openclaw/* /tmp/openclaw-sandbox-new/.openclaw/
```

**问题**:
- 沙盒继承了生产的 `plugins.allow: ['lossless-claw']`
- 沙盒共享了生产的 `~/.openclaw/extensions/` 目录
- 两个 Gateway 同时尝试加载有问题的插件

**正确做法**:
```bash
# ✅ 创建全新的独立配置
mkdir -p /tmp/sandbox/.openclaw/{extensions,agents,logs}

cat > /tmp/sandbox/.openclaw/openclaw.json << 'EOF'
{
  "gateway": {"auth": {"mode": "token", "token": "sandbox-token"}},
  "plugins": {"allow": [], "entries": {}}  // 空插件列表
}
EOF
```

---

### 2. 修复时直接修改生产配置

**错误**:
```bash
# ❌ 直接修改 ~/.openclaw/openclaw.json
openclaw config unset plugins.entries.lossless-claw
```

**影响**: 同时影响生产环境和沙盒环境

**正确做法**:
```bash
# ✅ 只修改沙盒配置
OPENCLAW_HOME=/tmp/sandbox openclaw config unset plugins.entries.lossless-claw
```

---

### 3. 插件安装路径错误

**错误**:
```bash
# ❌ 创建软链接到不存在的路径
ln -s $(npm root -g)/@martian-engineering/lossless-claw ~/.openclaw/extensions/
```

**问题**: npm 全局安装位置不存在，软链接指向空目录

**正确做法**:
```bash
# ✅ 使用真实目录
git clone https://github.com/martian-engineering/lossless-claw.git ~/.openclaw/extensions/lossless-claw
cd ~/.openclaw/extensions/lossless-claw
npm install
```

## ✅ 沙盒隔离标准流程

### 步骤 1: 创建独立目录

```bash
SANDBOX_NAME="plugin-test-$(date +%Y%m%d)"
SANDBOX_PATH="/tmp/openclaw-sandbox-${SANDBOX_NAME}"

mkdir -p ${SANDBOX_PATH}/.openclaw/{extensions,agents/writer,logs,canvas}
```

### 步骤 2: 创建独立配置

```bash
cat > ${SANDBOX_PATH}/.openclaw/openclaw.json << 'EOF'
{
  "gateway": {
    "auth": {"mode": "token", "token": "sandbox-token-xxx"},
    "bind": "loopback",
    "port": 18796
  },
  "plugins": {
    "allow": [],
    "entries": {}
  },
  "meta": {
    "purpose": "插件测试沙盒",
    "isolated": true
  }
}
EOF
```

### 步骤 3: 在沙盒内安装插件

```bash
# 只复制到沙盒目录
cp -r /path/to/plugin ${SANDBOX_PATH}/.openclaw/extensions/

# 只修改沙盒配置
cat > ${SANDBOX_PATH}/.openclaw/openclaw.json << 'EOF'
{
  "plugins": {
    "allow": ["test-plugin"],
    "entries": {"test-plugin": {"enabled": true}}
  }
}
EOF
```

### 步骤 4: 启动沙盒 Gateway

```bash
OPENCLAW_HOME=${SANDBOX_PATH} openclaw gateway run --bind loopback --port 18796 &
```

### 步骤 5: 验证沙盒独立

```bash
# 检查沙盒插件列表
OPENCLAW_HOME=${SANDBOX_PATH} openclaw plugins list

# 检查生产插件列表（应不受影响）
openclaw plugins list
```

### 步骤 6: 测试完成后清理

```bash
pkill -f "OPENCLAW_HOME=${SANDBOX_PATH}"
rm -rf ${SANDBOX_PATH}
```

## 📝 关键原则

### 三不复制

1. **不复制生产配置** - 沙盒用全新配置
2. **不复制生产插件** - 沙盒独立安装测试插件
3. **不复制生产数据** - 沙盒用独立数据目录

### 三要独立

1. **配置独立** - `openclaw.json` 不共享
2. **插件独立** - `extensions/` 目录不共享
3. **端口独立** - Gateway 用不同端口（18796+）

### 测试流程

```
创建沙盒 → 安装插件 → 测试功能 → 验证通过 → 应用到生产 → 清理沙盒
                                    ↓
                              测试失败 → 直接丢弃沙盒
```

## 🛠️ 快速创建沙盒脚本

保存为 `~/.openclaw/scripts/create-sandbox.sh`:

```bash
#!/bin/bash
set -e

SANDBOX_NAME="${1:-test}"
SANDBOX_PATH="/tmp/openclaw-sandbox-${SANDBOX_NAME}"
PORT="${2:-18796}"
TOKEN="sandbox-${SANDBOX_NAME}-$(date +%s)"

echo "🔧 创建沙盒：${SANDBOX_NAME}"
echo "📁 路径：${SANDBOX_PATH}"
echo "🔌 端口：${PORT}"

# 创建目录
mkdir -p ${SANDBOX_PATH}/.openclaw/{extensions,agents/writer,logs,canvas}

# 创建配置
cat > ${SANDBOX_PATH}/.openclaw/openclaw.json << EOF
{
  "gateway": {
    "auth": {"mode": "token", "token": "${TOKEN}"},
    "bind": "loopback",
    "port": ${PORT}
  },
  "plugins": {"allow": [], "entries": {}},
  "meta": {"purpose": "沙盒测试", "isolated": true}
}
EOF

echo "✅ 沙盒创建完成"
echo "🌐 访问地址：http://127.0.0.1:${PORT}/#token=${TOKEN}"
echo ""
echo "启动沙盒："
echo "  OPENCLAW_HOME=${SANDBOX_PATH} openclaw gateway run --bind loopback --port ${PORT}"
echo ""
echo "清理沙盒："
echo "  pkill -f \"OPENCLAW_HOME=${SANDBOX_PATH}\""
echo "  rm -rf ${SANDBOX_PATH}"
```

使用方式：
```bash
# 创建沙盒
bash ~/.openclaw/scripts/create-sandbox.sh plugin-test 18796

# 启动沙盒
OPENCLAW_HOME=/tmp/openclaw-sandbox-plugin-test openclaw gateway run --bind loopback --port 18796
```

## 🎯 本次教训总结

| 问题 | 影响 | 改进 |
|------|------|------|
| 沙盒复制生产配置 | 两个环境同时崩溃 | 沙盒用全新配置 |
| 共享插件目录 | 插件错误影响生产 | 沙盒独立插件目录 |
| 直接修改生产配置 | 修复操作影响生产 | 使用 OPENCLAW_HOME 指定沙盒 |
| 软链接到不存在路径 | Gateway 崩溃 | 使用真实目录 |

---

**核心原则**: 沙盒测试必须完全隔离，任何测试失败都不应影响生产环境！
