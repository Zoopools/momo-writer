# Agency Agents 知识库评估计划

**来源**: https://github.com/msitarzewski/agency-agents  
**策略**: 分批读取，避免上下文溢出  
**Token 预算**: 每次读取 < 50K tokens

---

## 评估原则

### 安全策略
- ✅ 每次只读取 1-2 个 Agent 角色
- ✅ 提取核心信息后暂停
- ✅ 分批沉淀到知识库
- ✅ 遇到 400 错误立即停止

### 优先级排序
1. **高优先级**: 与现有 Agent 相关（writer/media/hunter）
2. **中优先级**: 可扩展的新领域
3. **低优先级**: 暂时用不到的领域

## 风险控制

### 停止条件
- Token 使用 > 100K
- 遇到 400 错误
- 读取时间 > 5 分钟
- 内容重复或无价值

### 备份策略
- 每批读取后立即提交 Git
- 关键发现写入 MEMORY.md
- 避免重复读取同一文件
