## 🎯 核心进化点

### 1. 配置加载器演进史

#### 问题链
```
环境变量异常 → PATH 找不到 cat/jq
↓
权限过度限制 → 700/600 导致 permission denied
↓
复杂缓存逻辑 → init_config_cache 变量冲突
↓
Zsh 解析错乱 → heredoc/here-strings 失败
```

#### 最终方案
```bash
# 极简版 config-loader.sh（放弃缓存，直接读取）
load_config() {
    local mod="$1"
    local key="$2"
    local file="$CONFIG_DIR/$mod.json"
    chmod 644 "$file" 2>/dev/null
    /usr/local/bin/jq -r ".$key // empty" "$file" 2>/dev/null
}
```

#### 进化价值
- ✅ **稳定性 > 性能**：纳秒级缓存 vs 毫秒级直读，选择可靠
- ✅ **简单 > 复杂**：少即是多，减少出错点
- ✅ **兼容 > 优化**：适配所有环境，不做特殊假设

### 2. 权限管理演进

#### 初始设计
```bash
chmod 700 ~/.openclaw/config          # 仅当前用户可访问
chmod 600 ~/.openclaw/config/**/*.json # 仅当前用户可读写
```

#### 实际问题
- macOS/Zsh 环境下脚本无法读取配置文件
- 子进程继承权限导致 permission denied

#### 最终方案
```bash
chmod 755 ~/.openclaw/config          # 当前用户可写，其他人可读
chmod 644 ~/.openclaw/config/**/*.json # 当前用户可写，其他人可读
```

#### 进化价值
- ✅ **安全平衡**：单用户系统无需过度防护
- ✅ **实用主义**：能用的权限才是好权限
- ✅ **文档化**：在 ARCHIVE.md 中明确记录权限选择

### 3. 脚本执行模式演进

#### 问题
```bash
# heredoc 在 Zsh 中易卡住
cat > file << 'EOF'
...
EOF

# 注释被 Zsh 当成命令
# 这是一个注释 → zsh: command not found: #
```

#### 解决方案
1. **分块写入**：大文件分成多个 cat 追加
2. **避免注释**：脚本中不用 `#` 开头
3. **使用 echo**：简单内容用 echo 代替 cat

#### 进化价值
- ✅ **环境感知**：不假设 shell 行为一致
- ✅ **防御性编程**：预设最坏情况
- ✅ **可复制性**：确保终端复制粘贴 100% 成功
