## 🏗️ 混合层级隔离架构

### 架构设计

```
哥哥
├── 墨墨 (writer) - 首席协调员（主要对话窗口，90% 任务）
│   ├── 职责：任务分配、审查、知识管理、协调各 Agent
│   ├── 模型：bailian/qwen3.5-plus
│   └── 限制：禁止执行 baoyu-* 技能（SOUL.md 约束）
│
└── 小媒 (media) - 创意专家（备用直接窗口，10% 任务）
    ├── 职责：图片生成、内容创作、新媒体发布
    ├── 模型：bailian/glm-4.7
    └── 权限：独占使用 baoyu-* 技能（SOUL.md 授权）
```

### 任务流转规范

| 任务类型 | 推荐路径 | 说明 |
|---------|---------|------|
| **复杂任务** | 哥哥→墨墨→小媒 | 墨墨统筹协调 |
| **简单图片** | 哥哥→小媒 | 直接快速 |
| **新媒体发布** | 哥哥→墨墨→小媒 | 墨墨审查质量 |
| **紧急任务** | 哥哥→小媒 | 快速响应 |
| **学习/讨论** | 哥哥↔小媒 | 直接沟通 |
| **多 Agent 协作** | 哥哥→墨墨→多 Agent | 墨墨协调 |

## 📦 baoyu-skills 隔离配置

### 文件位置

| 项目 | 路径 | 说明 |
|------|------|------|
| **专属目录** | `~/.openclaw/agents/media/skills/baoyu-skills/` | 小媒专属技能 |
| **软链接** | `~/.openclaw/skills/baoyu-skills` → 专属目录 | 网关加载用 |
| **墨墨 SOUL.md** | `~/Documents/openclaw/agents/writer/SOUL.md` | 权限限制 |
| **小媒 SOUL.md** | `~/Documents/openclaw/agents/media/SOUL.md` | 权限授权 |
| **API 配置** | `~/Documents/openclaw/agents/media/.env.local` | DashScope API |

### 隔离原理

1. **物理隔离**: baoyu-skills 在专属目录，通过软链接到全局
2. **逻辑隔离**: SOUL.md 文件注入权限规则（不修改 JSON）
3. **网关加载**: 软链接让网关注册技能名
4. **版本兼容**: 2026.3.2 不支持 systemPrompt，使用 SOUL.md 是正确方案

### 检查命令

```bash
# 检查软链接
ls -la ~/.openclaw/skills/ | grep baoyu

# 检查 SOUL.md
cat ~/Documents/openclaw/agents/writer/SOUL.md
cat ~/Documents/openclaw/agents/media/SOUL.md

# 配置验证
openclaw config validate

# Gateway 状态
openclaw status

# 检查 API 配置
cat ~/Documents/openclaw/agents/media/.env.local
```

## 🔧 故障排查

### 问题 1：技能不加载

**症状**: `openclaw status` 看不到 baoyu 技能

**检查**:
```bash
# 检查软链接是否有效
ls -la ~/.openclaw/skills/baoyu-skills

# 检查专属目录是否存在
ls -la ~/.openclaw/agents/media/skills/baoyu-skills/
```

**解决**:
```bash
# 如果软链接失效，重新创建
rm -f ~/.openclaw/skills/baoyu-skills
ln -s ~/.openclaw/agents/media/skills/baoyu-skills ~/.openclaw/skills/baoyu-skills

# 重启网关
openclaw gateway restart --force
```

---

### 问题 2：配置报错

**症状**: `openclaw config validate` 报错 "Unrecognized key"

**原因**: openclaw.json 包含不支持的字段（如 systemPrompt）

**解决**:
```bash
# 恢复备份
cp ~/.openclaw/openclaw.json.backup.* ~/.openclaw/openclaw.json

# 重启网关
openclaw gateway restart --force

# 使用 SOUL.md 而非 JSON 注入权限
```

---

### 问题 3：权限不生效

**症状**: 墨墨可以执行 baoyu 技能，或小媒无法执行

**检查**:
```bash
# 检查 SOUL.md 内容
cat ~/Documents/openclaw/agents/writer/SOUL.md
cat ~/Documents/openclaw/agents/media/SOUL.md
```

**解决**:
```bash
# 重新创建 SOUL.md（参考下方模板）
# 重启网关
openclaw gateway restart --force
```

---

### 问题 4：Gateway 无法启动

**症状**: `openclaw gateway start` 失败，1006 错误

**解决**:
```bash
# 1. 强制杀掉残留进程
ps aux | grep openclaw | grep -v grep | awk '{print $2}' | xargs kill -9 2>/dev/null

# 2. 清理锁文件
rm -f ~/.openclaw/gateway.lock
rm -rf ~/.openclaw/run/* ~/.openclaw/tmp/*

# 3. 恢复备份
cp ~/.openclaw/openclaw.json.backup.* ~/.openclaw/openclaw.json

# 4. 冷启动
openclaw gateway start --force --allow-unconfigured
```

## 📝 SOUL.md 模板

### 墨墨的 SOUL.md

```markdown
# 墨墨 (Mò) - 首席协调员

## 核心限制
- ❌ 禁止直接执行 /agents/media/skills/ 目录下的所有工具。
- ❌ 禁止调用任何 baoyu-* 开头的技能。
- ✅ 仅允许读取这些工具的输出结果进行质量审查。

## 协作规则
- 发现绘图或新媒体生成需求，必须指派给小媒 (media) 执行。
```

### 小媒的 SOUL.md

```markdown
# 小媒 (Xiǎo Méi) - 创意专家

## 核心权限
- ✅ 你拥有执行 /agents/media/skills/ 下 baoyu-* 技能的独占权。
- ✅ 你直接对哥哥负责，同时接受墨墨的任务分配。

## 工作流
- 完成 baoyu 任务后，请在 /memory/sync.md 记录执行简报，方便墨墨同步审查。
```
