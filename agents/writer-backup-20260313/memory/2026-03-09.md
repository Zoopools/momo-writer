## 🕐 13:16 PM - Lossless Claw 插件测试失败与沙盒隔离教训 ⭐⭐⭐⭐⭐

### 事件概述

**目标**: 测试 Lossless Claw 插件（LCM 上下文管理系统）  
**结果**: ❌ 失败 - 插件与 OpenClaw 2026.3.2 API 不兼容  
**影响**: Gateway 崩溃，WebUI 无法访问（已恢复）  
**核心教训**: 沙盒隔离缺失导致测试影响生产环境

---

### 时间线

| 时间 | 操作 | 状态 |
|------|------|------|
| 12:36 PM | npm install -g @martian-engineering/lossless-claw | ✅ 成功 |

| 12:52 PM | 创建沙盒（复制生产配置） | ❌ **错误起点** |
| 12:55 PM | 两个 Gateway 同时加载 lossless-claw | ❌ 都失败 |
| 12:58 PM | 尝试软链接修复 | ❌ Gateway 崩溃 |
| 01:07 PM | 删除插件配置 + 重启 | ✅ 生产恢复 |
| 01:14 PM | 创建隔离沙盒 + 记录教训 | ✅ 完成 |

---

### 核心错误

#### 1. 沙盒复制生产配置 ❌

```bash
# 错误做法
cp -r ~/.openclaw/* /tmp/openclaw-sandbox-new/.openclaw/
```

**问题**:
- 沙盒继承了 `plugins.allow: ['lossless-claw']`
- 沙盒共享了 `~/.openclaw/extensions/`
- 两个 Gateway 同时尝试加载有问题的插件

#### 2. API 不兼容

```
TypeError: api.registerContextEngine is not a function
```

**原因**: lossless-claw 调用旧版 API，OpenClaw 2026.3.2 已移除该方法

#### 3. 修复时直接影响生产

```bash
# 直接修改生产配置
openclaw config unset plugins.entries.lossless-claw
```

**影响**: 生产环境和沙盒同时被修改

---

### 解决方案

#### 1. 删除插件配置

```bash
openclaw config unset plugins.entries.lossless-claw
openclaw config set plugins.allow '["feishu-openclaw-plugin"]'
rm -rf ~/.openclaw/extensions/lossless-claw
```

#### 2. 重启 Gateway

```bash
openclaw gateway stop
pkill -f openclaw-gateway
openclaw gateway install
openclaw gateway start
```

#### 3. 验证恢复

```bash
openclaw status          # ✅ Gateway 运行中 (PID 79136)
openclaw plugins list    # ✅ lossless-claw 已移除
```

---

### 正确沙盒隔离流程

#### 创建独立沙盒

```bash
# 1. 创建独立目录（不复制生产）
mkdir -p /tmp/openclaw-sandbox-isolated/.openclaw/{extensions,agents,logs}

# 2. 创建全新配置（空插件列表）
cat > /tmp/openclaw-sandbox-isolated/.openclaw/openclaw.json << 'EOF'
{
  "gateway": {
    "auth": {"mode": "token", "token": "sandbox-token-xxx"},
    "port": 18796
  },
  "plugins": {"allow": [], "entries": {}}
}
EOF

# 3. 启动沙盒
OPENCLAW_HOME=/tmp/openclaw-sandbox-isolated openclaw gateway run --bind loopback --port 18796
```

#### 三不复制原则

1. ❌ 不复制生产配置
2. ❌ 不复制生产插件
3. ❌ 不复制生产数据

#### 三要独立

1. ✅ 配置独立
2. ✅ 插件独立
3. ✅ 端口独立

## 🚀 17:26 PM - 沙盒 Gateway 终极复活！⭐⭐⭐⭐⭐

### **核心突破**

沙盒 Gateway 成功启动并稳定运行超过 30 分钟！这是经过豆包 + Gemini 联合诊断后的终极解决方案。

### **根因分析**（豆包 + Gemini 联合诊断）

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| **进程秒退** | macOS 回收低优先级进程 (SN 状态) + 空闲超时 | nohup 后台运行 |
| **参数错误** | `--no-bonjour` 在 2026.3.8 已废弃 | 移除该参数 |
| **配置无效** | `gateway.bonjour` 和 `agents.defaults.heartbeat.enabled` 不存在 | jq 删除无效配置键 |
| **ID 冲突** | 沙盒与生产 agent-id 一致触发脑裂保护 | 修改为 writer-sandbox / media-sandbox |
| **CORS 错误** | allowedOrigins 残留生产端口 18789 | jq 同步为 18798 |
| **QMD 内存峰值** | 全量索引扫描导致内存飙升 | 关闭 memorySearch |

### **配置修复**

```bash
# 删除无效配置键（2026.3.8 不支持）
jq 'del(.gateway.bonjour) | del(.agents.defaults.heartbeat.enabled)' openclaw.json

# 修复 allowedOrigins + 唯一化 agent-id + 关闭 memorySearch
jq '
 .gateway.controlUi.allowedOrigins = ["http://127.0.0.1:18798", "http://localhost:18798"] |
 .agents.list[0].id = "writer-sandbox" |
 .agents.list[1].id = "media-sandbox" |
 .agents.defaults.memorySearch.enabled = false
' openclaw.json
```

### **成功启动命令**

```bash
cd /tmp/openclaw-upgrade-sandbox
export OPENCLAW_HOME="/tmp/openclaw-upgrade-sandbox"
export HOME="/tmp/openclaw-upgrade-sandbox"
nohup node node_modules/openclaw/openclaw.mjs gateway run \
  --bind loopback \
  --port 18798 \
  > .openclaw/logs/gateway-latest.log 2>&1 &
```

```bash
# 生产环境专用
```

1. ✅ 自动清理环境变量 (`unset OPENCLAW_HOME`)

3. ✅ 彩色视觉提示 (生产=绿色，沙盒=黄色)
4. ✅ 版本显示 (自动显示 OpenClaw 版本)
5. ✅ 存在检查 (沙盒不存在时提前报错)

## 📋 重要教训

### 环境隔离原则

1. **永远使用专用脚本** - 不手动设置环境变量
2. **端口必须隔离** - 生产 18789，沙盒 18800
3. **配置不共享** - 沙盒配置独立于生产
4. **视觉区分** - 颜色提示当前环境

| 配置项 | 2026.3.2 | 2026.3.8 |

| `gateway.log` | ✅ 支持 | ❌ 已移除 |
| 绑定模式 | IP 或模式名 | 仅模式名 (loopback) |
