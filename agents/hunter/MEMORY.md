## 重要经验

### 信息抓取类项目
- 优先使用官方 API（稳定）
- 设置合理的 User-Agent
- 添加请求间隔（避免被封）
- 处理异常情况（网络错误、数据缺失）

### GitHub Pages 部署
- 使用英文文件名（避免编码问题）
- 设置 `index.html` 作为首页
- 启用自动部署（push 后自动更新）
- 使用相对路径（避免路径错误）

### HTML 报告设计
- 渐变背景（视觉吸引力）
- 卡片布局（信息分组清晰）
- 悬停动画（交互反馈）
- 响应式设计（多设备兼容）
