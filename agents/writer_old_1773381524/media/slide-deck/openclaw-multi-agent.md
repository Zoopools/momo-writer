# OpenClaw 多 Agent 架构

## 什么是多 Agent 系统？

OpenClaw 的多 Agent 系统是一个协同工作的智能体网络，每个 Agent 专注于特定领域，通过 Gateway 进行通信和协调。

## 为什么需要多 Agent？

### 1. 专业化分工
- 每个Agent专注特定领域（如小媒专注新媒体、专家Agent专注技术开发）
- 避免单Agent过度复杂化
- 提高任务完成效率

### 2. 灵活扩展
- 轻松添加新Agent能力
- 动态路由任务到合适Agent
- 支持插件化架构

### 3. 并行协作
- 多个Agent可同时工作
- 加速复杂任务完成
- 支持任务分解与分配

## Agent 通信机制

### 1. 会话路由
- Gateway 根据消息内容选择目标Agent
- 支持显式路由（指定Agent）和自动路由（内容分析）
- 维护跨Agent会话历史

### 2. 消息格式
```json
{
  "message_id": "unique_id",
  "from_session": "sender_session_key",
  "to_session": "target_session_key",
  "content": "消息内容",
  "context": {
    "user_id": "ou_xxx",
    "channel": "discord",
    "timestamp": "2026-03-09T07:00:00Z"
  }
}
```

### 3. 上下文传递
- 自动传递必要的上下文信息
- 支持会话历史追溯
- 保持对话连贯性

## 任务分配与协调

### 1. 任务分解
- 复杂任务自动分解为子任务
- 每个子任务分配给最合适的Agent
- 支持并行执行和结果聚合

### 2. 协调模式
- **串行协调**：按顺序执行（A → B → C）
- **并行协调**：同时执行多个任务
- **混合协调**：部分并行，部分串行

### 3. 结果聚合
- 收集所有Agent的执行结果
- 合并为统一响应
- 返回给用户

## 实际应用场景

### 场景 1：内容创作工作流
```
用户请求 → 墨墨（分析需求）
    ↓
    → 小媒（创作内容）
    ↓
    → 专家Agent（技术审查）
    ↓
墨墨（整合结果） → 用户
```

### 场景 2：多平台发布
```
用户请求 → 小媒（创作内容）
    ↓
    → 多个发布Agent（微信/微博/X）
    ↓
小媒（收集反馈） → 用户
```

### 场景 3：问题排查
```
用户问题 → Gateway（路由到专家）
    ↓
    专家Agent（诊断）
    ↓
    → 如果需要 → 小媒（用户沟通）
    ↓
专家Agent（解决方案） → 用户
```

## 开始使用

### 1. 创建Agent
```bash
# 创建新的Agent配置
openclaw agent create my-agent
```

### 2. 定义技能
```bash
# 在Agent目录下创建技能
mkdir -p ~/.openclaw/agents/my-agent/skills/my-skill
```

### 3. 配置路由
```json
{
  "agents": {
    "media": {
      "name": "小媒",
      "skills": ["baoyu-*"]
    }
  }
}
```

### 4. 测试通信
```bash
# 向特定Agent发送消息
openclaw send --to media "你好小媒"
```
