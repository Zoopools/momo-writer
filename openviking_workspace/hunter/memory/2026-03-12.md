# 2026-03-12 - B 站评论抓取项目进展

## CDP 浏览器配置成功 ✅

**配置步骤：**
1. 关闭 Chrome：`pkill -f "Google Chrome"`
2. 复制用户数据目录：`cp -r ~/Library/Application\ Support/Google/Chrome ~/Library/Application\ Support/Google/ChromeDebug`
3. 启动 CDP Chrome：
   ```bash
   /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
     --remote-debugging-port=9222 \
     --remote-debugging-address=127.0.0.1 \
     --user-data-dir="$HOME/Library/Application Support/Google/ChromeDebug"
   ```
4. 配置 OpenClaw `~/.openclaw/openclaw.json`：
   ```json
   {
     "browser": {
       "defaultProfile": "my-browser",
       "profiles": {
         "my-browser": {
           "cdpUrl": "http://127.0.0.1:9222",
           "color": "#0066CC"
         }
       }
     }
   }
   ```
5. 创建启动脚本：`/Users/wh1ko/Documents/openclaw/agents/hunter/start-chrome-debug.sh`

## 当前问题 ⚠️

**CDP 浏览器没有 B 站登录状态**
- CDP 浏览器是复制的用户数据目录快照
- 复制时可能没有包含 B 站登录 Cookie
- 需要手动在 CDP 浏览器中登录 B 站

## 解决方案

1. **手动登录**：在 Chrome Debug 浏览器访问 B 站并登录
2. **或使用主浏览器 Cookie**：从已登录的 Chrome 复制 Cookie 注入

## 参考文章

关于 OpenClaw CDP 浏览器配置的详细教程：
- 原理：通过 Chrome DevTools Protocol 直接连接
- 优势：使用真实浏览器登录状态，比扩展中继更稳定
- 限制：user-data-dir 必须是非默认路径
