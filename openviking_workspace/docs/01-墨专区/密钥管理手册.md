# 🔐 密钥管理手册

> **协议版本**：3.13 全域合体  
> **生效日期**：2026-03-16  
> **管理权**：墨墨（Matrix 首席密钥官）

---

## 一、协议概述

### 首席密钥官任命

**墨墨（Writer Agent）** 正式接任 **Matrix 首席密钥官**，全权负责所有 API Token、Cookie 及身份凭证的维护与安全。

### 核心原则

1. **集中管理**：所有密钥统一存储于 `~/.media/config.yaml`
2. **双端对齐**：家/公司两端路径及格式完全一致
3. **安全隔离**：严禁将真实密钥提交到 Git
4. **权限分级**：墨墨管理，小媒/小猎申请使用

---

## 二、物理路径

```
~/.media/
├── config.yaml              # 🔴 真实密钥文件（Git 忽略）
├── config.yaml.template     # 🟢 模板文件（Git 同步）
└── .gitignore              # 排除规则
```

### 文件权限

```bash
chmod 600 ~/.media/config.yaml      # 仅所有者可读写
chmod 644 ~/.media/config.yaml.template  # 所有人可读
```

---

## 三、调用规范

### 墨墨职责
- 编写 `key-manager` 密钥调用接口
- 提供 `config.yaml.template` 模板
- 维护密钥版本与更新

### 小媒 & 小猎规范
- ❌ **禁止** 私自存储 Token
- ❌ **禁止** 在代码中硬编码密钥
- ✅ **必须** 通过墨墨定义的路径检索获取

### 使用示例

```bash
# 获取 Twitter Cookie
key-manager get twitter cookie

# 获取 GitHub Token
key-manager get github token

# 获取 Bilibili Cookie
key-manager get bilibili cookie
```

---

## 四、安全红线

| 红线 | 违规后果 |
|------|---------|
| 🔴 严禁将真实 `config.yaml` 推送到远程仓库 | 密钥泄露，账号风险 |
| 🔴 严禁在代码中硬编码 Token | 代码泄露，密钥暴露 |
| 🔴 严禁私自共享密钥给其他 Agent | 权限混乱，安全失控 |
| 🟡 定期更换 Cookie（建议 30 天）| 降低被封风险 |
| 🟢 使用专用小号登录平台 | 保护主号安全 |

## 六、应急处理

### 密钥泄露

1. **立即撤销**：在对应平台撤销泄露的 Token
2. **生成新密钥**：重新生成并配置
3. **更新配置**：替换 `~/.media/config.yaml`
4. **同步双端**：确保家/公司都更新
5. **审查日志**：检查是否有异常使用

### 同步失败

```bash
# 手动同步密钥配置
key-manager sync

# 验证配置
key-manager status
```
