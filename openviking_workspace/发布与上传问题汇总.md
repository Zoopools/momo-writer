3. **记录成功的命令** - 下次直接用

### 问题 4: 设备认证问题

**出现次数**: 1 次 (2026-03-08 15:19)

**错误**:
- 墨墨尝试用 operator token 发布
- 返回 401 authentication_required
- 实际应该用 X-Device-ID header

**正确做法**:
```bash
# 获取设备 ID
cat ~/.openclaw/identity/device-auth.json | jq -r '.deviceId'

# 用设备 ID 认证
curl -X POST ... -H "X-Device-ID: <设备 ID>" ...
```

**教训**:
1. **operator token ≠ 发布权限** - 需要设备认证
2. **设备 ID 在 device-auth.json** - 不是 device.json
3. **X-Device-ID header** - 不是 Authorization Bearer

**墨墨的承诺**:
> "水产市场发布用 X-Device-ID header，不是 Authorization！"

## 📝 标准发布流程 (墨墨必记！)

### 水产市场发布流程

```bash
# 1. 准备文件
cd /path/to/skill
zip -r /tmp/skill.zip ./*.md

# 2. 获取设备 ID
DEVICE_ID=$(cat ~/.openclaw/identity/device-auth.json | jq -r '.deviceId')

# 3. curl API 发布
curl -X POST "https://openclawmp.cc/api/v1/assets/publish" \
  -H "X-Device-ID: $DEVICE_ID" \
  -F "package=@/tmp/skill.zip" \
  -F 'metadata={
    "name": "Skill 名称",
    "type": "skill",
    "version": "1.0.0",
    "description": "简短描述",
    "tags": ["tag1", "tag2"]
  }' | jq '.'

# 4. 检查返回
# {"success":true,"data":{"id":"s-xxx",...}} → 成功！
```

**注意事项**:
- ✅ 用 curl API，不用 openclawmp publish CLI
- ✅ X-Device-ID header，不是 Authorization
- ✅ tags 用英文
- ✅ metadata 是 JSON 字符串

### ClawHub 发布流程

```bash
# 1. 准备文件
cd /path/to/skill

# 2. clawhub CLI 发布
clawhub publish \
  --slug "skill-slug" \
  --name "Skill 名称" \
  --version "1.0.0" \
  --changelog "更新内容" \
  --tags "tag1,tag2,tag3" \  # ← 英文！
  .

# 3. 检查返回
# ✔ OK. Published skill@version (k97...) → 成功！
```

**注意事项**:
- ✅ tags 只能用英文和连字符
- ✅ --slug 用小写和连字符
- ✅ 当前目录用 `.`

---

### GitHub 发布流程

```bash
# 1. 检查 GitHub 用户名
GITHUB_USER=$(gh api user | jq -r '.login')
echo "GitHub 用户名：$GITHUB_USER"

# 2. 准备仓库
cd /path/to/skill
git init
git add .
git commit -m "v1.0.0: 描述"

# 3. 创建并推送
gh repo create $GITHUB_USER/repo-name \
  --public \
  --description "描述" \
  --source=. \
  --remote=origin \
  --push

# 或者手动设置 remote
git remote add origin git@github.com:$GITHUB_USER/repo-name.git
git push -u origin main
```

**注意事项**:
- ✅ 先用 gh api user 确认用户名
- ✅ gh repo create 自动设置 remote
- ✅ Public 仓库
