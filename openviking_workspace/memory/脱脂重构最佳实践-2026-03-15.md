## 🎯 什么是"脱脂重构"

> **定义**: 从 Git 追踪中剔除环境依赖和临时文件，只保留核心思想和资产，让同步链路传输真正有价值的内容。

**核心理念**:
- **轻** = 同步快、结构清、易维护
- **重** = 资产厚、产出多、价值高
- **传输思想，不是耗材**

---

|------|--------|--------|----------|

| 同步时间 | 慢 | 快 | **大幅提升** |
| 仓库清晰度 | 混杂 | 清晰 | **一眼可见核心** |

### 步骤 2: 清理 Git 缓存

```bash
# 1. 进入每个 Agent 目录
cd ~/Documents/openclaw/agents/writer
cd ~/Documents/openclaw/agents/hunter
cd ~/Documents/openclaw/agents/media

# 2. 从 Git 缓存中移除已追踪的依赖文件（但保留本地文件）
git rm -r --cached node_modules/
git rm -r --cached __pycache__/
git rm --cached package-lock.json

# 3. 提交变更
git add .gitignore
git commit -m "脱脂重构：剔除环境依赖，只保留核心资产"

# 4. 推送到远程
git push origin main
```

### 步骤 3: 资产分类标准

#### 核心资产（必须保留）

| 类别 | 示例 | 原因 |
|------|------|------|
| **人格配置** | SOUL.md, IDENTITY.md, MEMORY.md | 定义 Agent 是谁 |
| **任务记录** | 2026-03-15.md, 工作日志 | 记录做了什么 |
| **创作产出** | 文章、幻灯片、封面图 | 有价值的成果 |
| **脚本工具** | daily-briefing.sh, evolve-auto.sh | 自动化能力 |
| **知识库** | knowledge/*.md, 经验总结 | 沉淀的智慧 |

#### 环境耗材（应该剔除）

| 类别 | 示例 | 原因 |
|------|------|------|
| **依赖包** | node_modules/, __pycache__/ | 可重新安装 |
| **锁文件** | package-lock.json, bun.lock | 环境相关 |
| **临时文件** | *.tmp, *.log, .DS_Store | 无价值 |
| **大型数据** | *.csv, *.json.gz | 占用空间 |
| **备份文件** | *.backup.* | 可本地保留 |

## 🎯 关键原则

1. **传输思想，不是耗材**
   - 依赖可以重新安装，思想不能丢失

2. **核心资产优先**
   - 人格配置 > 任务记录 > 创作产出 > 脚本工具

3. **环境可重建**
   - node_modules/ 和 __pycache__/ 随时可重建，无需追踪

4. **定期脱脂**
   - 建议每月检查一次，防止环境脂肪重新堆积
