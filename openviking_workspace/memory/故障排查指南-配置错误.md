# 故障排查指南 - 配置错误导致 Gateway 无法启动

**创建日期：** 2026-03-07  
**触发事件：** 墨墨尝试配置权限字段导致配置无效  
**OpenClaw 版本：** 2026.3.2

---

## 🚨 故障现象

### 典型症状

1. **配置验证失败**
   ```
   Invalid config at /Users/wh1ko/.openclaw/openclaw.json:
   - agents.defaults: Unrecognized key: "timeout"
   - agents.list.0: Unrecognized keys: "timeout", "autoApprove", "permissions"
   ```

2. **Gateway 无法启动**
   ```
   Gateway start blocked: set gateway.mode=local
   ```

3. **WebUI 无法访问**
   - http://127.0.0.1:18789 打不开
   - 或者加载后无法连接

4. **CPU 占用 100%**
   - Gateway 进程循环加载
   - 系统发热、风扇狂转

5. **飞书机器人无法回复**
   - 消息发送后无响应
   - 日志显示配置错误

---

## 🔍 诊断流程

### 步骤 1：检查配置状态

```bash
openclaw config validate
```

**可能结果：**
- ✅ `Config valid` - 配置正常
- ❌ `Unrecognized key` - 配置有无效字段

### 步骤 2：运行 Doctor 诊断

```bash
openclaw doctor
```

**查看输出：**
- Config 部分是否显示 "Config invalid"
- Doctor changes 部分列出需要修复的字段

### 步骤 3：检查 Gateway 状态

```bash
openclaw status
```

**关键指标：**
- Gateway service: running / not running
- Feishu: ON/OFF, OK/SETUP/ERROR
- CPU 占用：正常 / 100%

### 步骤 4：查看日志

```bash
tail -f ~/.openclaw/logs/gateway.log | grep -i error
```

**查找关键词：**
- `Unrecognized key`
- `Gateway start blocked`
- `plugin mismatch`

---

## 🛠️ 修复方案

### 方案 A：自动修复（推荐）

```bash
# 1. 自动修复配置
openclaw doctor --fix

# 2. 验证修复结果
openclaw config validate

# 3. 重新安装网关
openclaw gateway install --force

# 4. 启动网关
openclaw gateway start --force --allow-unconfigured

# 5. 验证状态
openclaw status
```

### 方案 B：手动修复

```bash
# 1. 备份当前配置
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak

# 2. 编辑配置，删除无效字段
vim ~/.openclaw/openclaw.json

# 3. 验证配置
openclaw config validate

# 4. 重启网关
openclaw gateway restart --force
```

### 方案 C：从备份恢复

```bash
# 1. 恢复备份配置
cp ~/.openclaw/openclaw.json.bak ~/.openclaw/openclaw.json

# 2. 验证配置
openclaw config validate

# 3. 重启网关
openclaw gateway restart --force
```

## ⚠️ 2026.3.2 版本不支持的字段

**以下字段会导致配置无效：**

| 字段 | 位置 | 状态 | 替代方案 |
|------|------|------|----------|
| `agents.defaults.timeout` | ❌ 不支持 | 使用默认值 |
| `agents.list[].timeout` | ❌ 不支持 | 使用默认值 |
| `agents.list[].autoApprove` | ❌ 不支持 | 人工确认 |
| `agents.list[].permissions` | ❌ 不支持 | Workspace 隔离 |
| `plugins.entries.feishu.id` | ❌ 不支持 | 忽略警告 |

**安全提示：**
- 这些字段在更新版本中可能支持
- 不要盲目复制 AI 生成的配置
- 先用 `openclaw config validate` 验证

## 💡 经验教训

### 本次故障的教训

1. **不要盲目相信 AI 建议**
   - Gemini/豆包的建议可能基于不同版本
   - 需要先验证是否适配当前版本

2. **配置修改前先验证**
   - 使用 `openclaw config validate`
   - 或者先用 `openclaw doctor` 检查

3. **备份是最后的防线**
   - 重要修改前一定要备份
   - 备份文件会自动生成，但手动备份更可靠

4. **理解版本差异**
   - 2026.3.2 是较旧版本
   - 新功能可能需要升级

5. **插件警告无需过度担心**
   - `plugin id mismatch` 通常是兼容性问题
   - 不影响核心功能可以忽略
