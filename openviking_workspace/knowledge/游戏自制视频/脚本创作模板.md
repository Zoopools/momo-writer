## 通用脚本结构

### 标准结构 (8 分钟视频)

```
0:00-0:05   黄金 5 秒钩子      (留住观众)
0:05-0:30   引入 + 预告       (建立期待)
0:30-2:00   要点 1            (核心内容)
2:00-4:00   要点 2            (核心内容)
4:00-6:00   要点 3            (核心内容)
6:00-7:00   高潮/亮点         (情绪峰值)
7:00-7:30   总结              (回顾要点)
7:30-8:00   引导 + 预告       (互动 + 关注)
```

### 时长分配原则

| 段落 | 占比 | 核心目标 |
|------|------|---------|
| 开头 (0-30 秒) | 6% | 留住观众 |
| 主体 (30 秒 -6 分钟) | 75% | 提供价值 |
| 结尾 (6-8 分钟) | 19% | 引导互动 |

## 攻略类脚本模板

### 适用场景
- 游戏通关攻略
- 角色培养指南
- 装备搭配推荐
- 技巧教学

### 脚本模板

```markdown
# 【游戏名】攻略脚本

## 开头 (0-30 秒)

**[钩子 - 5 秒]**
"你是不是也卡在这一关很久了？"
[画面：Boss 战失败画面]

**[痛点引入 - 10 秒]**
"试了无数次都过不去，装备也刷了，等级也够了，就是打不过"
[画面：玩家挫败表情/聊天截图]

**[解决方案 - 15 秒]**
"今天这个视频，我用 3 个技巧，帮你轻松通关"
[画面：通关成功画面 + 标题]

## 引入 (30 秒 -1 分钟)

**[背景介绍]**
"这个 Boss 是 XX 游戏的第 X 个 Boss，很多新手都卡在这里"
[画面：Boss 介绍/游戏地图]

**[难点分析]**
"它的难点主要有 3 个：
1. 攻击频率快
2. 技能范围大
3. 第二阶段会狂暴"
[画面：Boss 技能演示]

**[预告]**
"接下来我会逐一讲解应对方法"
[画面：目录/要点预览]

## 主体 (1-6 分钟)

**[要点 1: XX 技巧] (1-2:30 分钟)**
"第一个技巧是 XX，这是最关键的一步"
[画面：技巧演示]

"具体操作是这样的..."
[画面：慢动作分解 + 箭头标注]

"注意这里，很多人会犯的错误是..."
[画面：错误示范 vs 正确示范]

**[要点 2: XX 技巧] (2:30-4 分钟)**
"第二个技巧是 XX，配合第一个技巧使用"
[画面：技巧演示]

"这个技巧的关键在于时机把握"
[画面：时机提示 + 倒计时]

**[要点 3: XX 技巧] (4-6 分钟)**
"最后一个技巧是 XX，这是进阶玩法"
[画面：高阶技巧演示]

"掌握这个技巧后，你可以..."
[画面：实战应用]

## 高潮 (6-7 分钟)

**[完整演示]**
"现在我来完整演示一遍"
[画面：无剪辑通关过程]

"注意看这里的连招..."
[画面：关键操作特写]

## 结尾 (7-8 分钟)

**[总结]**
"总结一下，今天的 3 个技巧：
1. XX 技巧 - 核心要点
2. XX 技巧 - 配合使用
3. XX 技巧 - 进阶玩法"
[画面：要点回顾]

**[引导]**
"如果这个视频对你有帮助，记得点赞收藏"
[画面：点赞动画]

"有任何问题在评论区问我"
[画面：评论提示]

**[预告]**
"下期视频我会讲解 XX 攻略，记得关注"
[画面：下期预告]
```

## 爆款开头库

### 痛点型

```
"你是不是也遇到过..."
"为什么你总是..."
"别再 XX 了，这样才是对的"
"90% 的人都不知道的 XX 技巧"
```

### 悬念型

```
"没想到竟然是这样..."
"我发现了 XX 的秘密"
"官方不想让你知道的 XX"
"最后一个颠覆你的认知"
```

### 对比型

```
"之前 vs 现在，差距太大了"
"XX 元 vs XX 元，差距在哪里"
"新手 vs 老手，区别在这里"
"我试了 10 种方法，只有这个有效"
```

### 数字型

```
"3 个技巧帮你..."
"5 分钟学会..."
"10 个必看的..."
"XX 小时总结出来的经验"
```

### 冲突型

```
"我反对这个观点..."
"XX 游戏根本不配叫 XX"
"别再被 XX 骗了"
"这个攻略是错的"
```
